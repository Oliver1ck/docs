# Docs — расписание, учёт занятий и автоматические отчёты преподавателя

Веб-ресурс для преподавателей: ведение расписания занятий, фиксация факта их
проведения и, самое главное, **автоматическое формирование отчётов**, чтобы
избавить преподавателя от рутинного ручного подсчёта часов и занятий в конце
месяца.

## Главная задача — избавить от рутины отчётности

В конце месяца
преподавателю нужно вручную вспомнить и посчитать: сколько занятий было,
сколько из них состоялись, сколько часов отработано с каждым учеником или
группой, что отменено и по какой причине. Это муторная и подверженная ошибкам
работа, особенно при большом количестве учеников и разном расписании.

Проект берёт эту рутину на себя: преподаватель просто отмечает факт проведения
занятия (трекает), а система сама:

- считает количество проведённых занятий и суммарные часы за месяц —
  в целом и в разбивке по ученику/группе/предмету;
- показывает, какие занятия не состоялись и по какой причине (отмена, болезнь,
  перенос и т. д.);
- сверяет плановое расписание с фактическими треками и подсвечивает
  расхождения (не отмеченные занятия, пропуски);
- формирует готовый отчёт за период одним нажатием — без ручного сведения
  таблиц и заметок, с возможностью выгрузки/экспорта для отправки заказчику
  или в бухгалтерию.

## Ключевые сущности

1. **Расписание** — плановая сетка занятий (день недели, время, длительность,
   ученик/группа, предмет).
2. **Трек занятия** — фактическая отметка о проведённом (или не проведённом)
   занятии: дата, статус, длительность, комментарий. Это единственные данные,
   которые преподаватель вносит вручную день за днём.
3. **Отчёт** — автоматически собираемая на основе треков сводка за выбранный
   период (по умолчанию — месяц): количество и часы проведённых занятий,
   список несостоявшихся занятий и причины, итоги по каждому ученику/группе.
   Отчёт генерируется без дополнительных ручных действий преподавателя.

## Структура репозитория

```
.
├── backend/     Backend на Go — API, миграции БД, бизнес-логика
└── frontend/    Frontend на Vue 3 + TypeScript + Vite
```

### Backend (`backend/`)

- Язык: Go
- `cmd/api` — точка входа API-сервиса
- `internal` — внутренняя логика приложения
- `migrations` — миграции базы данных
- `pkg` — переиспользуемые пакеты
- `tests` — тесты
- `deploy` — конфигурация для развёртывания

### Frontend (`frontend/`)

- Vue 3 + TypeScript, сборка на Vite
- Пакетный менеджер: pnpm
- Линтинг/форматирование: oxlint, oxfmt

## Статус

Проект находится в начальной стадии разработки: заложена базовая структура
backend- и frontend-частей, основная функциональность (расписание, треки
занятий, отчёты) в разработке.

## Разработка

### Backend

```bash
cd backend
go run ./cmd/api
```

### Frontend

```bash
cd frontend
pnpm install
pnpm dev
```
